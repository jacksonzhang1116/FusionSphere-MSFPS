# GNSS 实时定位终端  
**K230 + GNSS + Web 可视化（支持 4G / MQTT 扩展）**

---

## 项目简介

本项目实现了一套 **低成本 GNSS 实时定位终端与可视化系统**。  
系统以 **01Studio CanMV K230 mini** 为核心控制单元，完成 GNSS 数据接收、解析与质量评估，并通过网络将定位结果实时回传至 Web 前端，在地图上进行动态展示。

项目目标是构建一个 **可独立运行、可工程扩展、具备产品化潜力的 GNSS 终端原型**，适用于室外定位展示、物联网终端、教学实验及导航感知系统前端。

## 软件与技术栈

### 嵌入式端（K230）

- 运行环境：MicroPython / CanMV 固件
- 功能：
  - UART 接收 GNSS NMEA 数据
  - 解析 GGA / GSA / RMC / GSV 等语句
  - 定位信息解算
  - 定位质量指标计算
  - JSON 格式输出

### PC 网关（Node.js）

- 串口通信：serialport
- 实时通信：WebSocket
- 功能：
  - 独占串口读取 K230 输出
  - JSON 解析与转发
  - 向 Web 前端实时推送定位数据

### Web 前端

- 地图引擎：百度地图 JavaScript API
- 坐标处理：WGS-84 → BD-09 实时转换
- 功能：
  - 实时定位点显示
  - 轨迹绘制
  - 定位质量面板（SV / SNR / DOP）
  - 连接状态与失联提示

# 运行模式说明（调试模式 / 演示模式）

本项目在实际使用中分为 **调试模式** 和 **演示模式** 两种运行方式。  
两种模式的核心区别在于：**串口由谁占用**。

> 串口资源在系统中是独占的，IDE 与 Node.js 服务器不可同时占用同一串口。

---

## 一、调试模式（开发 / 排错使用）

### 1. 使用目的

调试模式用于以下场景：

- 查看 GNSS 原始 NMEA 数据  
- 检查定位解算与 JSON 输出是否正确  
- 调试解析逻辑、参数计算过程  
- 使用 `print()` 输出中间变量进行排错  

---

### 2. 操作步骤

1. **不要启动 Node.js 服务器**

   ```
   确保没有运行 node server.js
   ```

2. **打开 CanMV IDE**
   - 连接 K230 设备
   - 打开串口监视器

3. **运行当前脚本**
   - 使用 IDE 的「运行当前脚本」
   - 不必保存为 `main.py`，便于反复修改调试

4. **在 IDE 串口中观察输出**
   - GNSS NMEA 语句
   - 解算后的 JSON 数据
   - 调试信息（如 GSV / DOP / 状态提示）

---

### 3. 注意事项

- IDE 串口仅用于**观察输出**，不是程序运行所必需  
- 调试完成后，如需上电自启动，必须保存为 `main.py`  
- 调试模式下不要启动 Node，否则会产生串口冲突  

---

### 4. 调试模式示意

```
GNSS → K230
         ↑
     IDE 串口监视器
```

---

## 二、演示模式（展示 / 实际运行）

### 1. 使用目的

演示模式用于以下场景：

- 地图实时显示定位结果  
- 展示轨迹、状态面板  
- 模拟真实终端运行状态  
- 项目演示、汇报或对外展示  

---

### 2. 操作步骤

1. **将稳定版本脚本保存为 `main.py`**
   - 在 CanMV IDE 中保存脚本到设备
   - 设为 `main.py`
   - 确认代码已测试稳定

2. **关闭 CanMV IDE**

   ```
   不要连接 IDE
   不要打开 IDE 串口
   ```

3. **给 K230 上电**
   - USB 或外部电源供电
   - 设备将自动运行 `main.py`

4. **启动 Node.js 服务器**

   ```
   node server.js
   ```

5. **打开 Web 前端页面**
   - 使用浏览器访问：
     ```
     http://localhost:xxxx
     ```
   - 地图将实时更新定位信息

---

### 3. 注意事项（非常重要）

- 演示模式下 **IDE 必须完全关闭**  
- Node.js 是此模式下 **唯一串口占用者**  
- 若 IDE 连接，会导致串口被抢占  

---

### 4. 演示模式示意

```
GNSS → K230（main.py 自动运行）
          ↓
        Node.js
          ↓
        浏览器地图
```

## 四、常见错误组合

### 错误 1：IDE 与 Node 同时运行
- 现象：IDE 显示“在忙碌”或 Node 打不开串口  
- 原因：串口资源被同时请求  

### 错误 2：忘记保存为 `main.py`
- 现象：设备重启后程序未运行  
- 原因：脚本未固化到自启动文件  

### 错误 3：使用 file:// 打开前端页面
- 现象：地图不显示或 AK 报错  
- 原因：地图 SDK 需通过 http/https 访问
